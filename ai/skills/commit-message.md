# Commit Message

When generating commit messages:

1. Use conventional commits format: `type(scope): description`
2. Types: feat, fix, refactor, docs, test, chore, perf, ci, build
3. Keep the subject line under 72 characters
4. Use imperative mood ("add" not "added")
5. Include a body if the change needs explanation (the "why", not the "what")
6. Reference issue numbers when available
